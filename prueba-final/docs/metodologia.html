Pasos seguidos para la creación de esta web

La última práctica de la asignatura persigue la finalidad de agrupar dentro de sí las demás que se han venido realizando para, finalmente, presentarlas todas en conjunto en una suerte de página web como esta. En primer lugar, descargamos Pandoc

para modificar el formato de los archivos que se encuentran en cada uno de mis repositorios (los cuales pasan así de MarkDown a HTML). Una vez integrada en el dispositivo mediante la introducción del comando "brew install pandoc", procedemos a la descarga de una carpeta de ejemplos: Bootstrap. Se descargaron cada uno de los ejemplos desde la terminal y, acto seguido, se descomprimió la carpeta para trabajar más fácilmente con ella. Una vez hecho esto, se crea una carpeta con el nombre de "Docs" y, con el comando mv, se trasladan index.html y sticky-footer.header.css a dicho emplazamiento. Al mismo tiempo, el archivo CSS se mueve, con "mkdir", a una carpeta también de nueva creación llamada CSS mediante "mv2". Dentro de Docs, creamos una carpeta nueva (js) y se traslada hasta allí el archibo bootstrap.bundle.min.js (javascript).

El siguiente paso consiste en crear varios archivos en HTML: la cabecera, el pie y todas las prácticas realizadas durante el curso (infografías primera y segunda, prácticas 3 y 4 y la presente metodología).

La edición de estos archivos se ha llevado a cabo desde nano, donde bajo el comando "Begin page content", se ha ido pegando el texto que correspondía (utilizando siempre el lenguaje HTML, obtenido mediante la traducción por medio de Pandoc).

Por otra parte, la cabecera se ha diseñado de modo que en esta apareciesen los debidos accesos directos a las distintas prácticas, las cuales se sitúan en diferentes páginas; después, se puede regresar al inicio cuando se considere (index.html).

El título de la página web ha sido modificado para identificarla como la última práctica del curso.

Por último, con el fin de que todo quedase subido en GitHub desde la terminal de cygwin, se han introducido ls comandos "git init" > "git add" > "git commit -m" y "git push"

Para conseguir la URL correspondiente a la página recién creada, accedí a mi cuenta de GitHub y desde la carpeta "Docs", fui a "Settings" y luego a "pages" y elegí la rama principal.

Tras actualizar la página, pude dar el proceso por finalizado.
